# Trek Spot Booking - Data Structure

## What Gets Saved to Database

When you submit the Trek Spot booking form, here's exactly what data is saved:

---

## 📋 Complete Data Structure

```javascript
{
  // Booking Identification
  bookingId: "TS-260209-001",  // Auto-generated: TS-YYMMDD-SEQ
  
  // Trek Spot Details (Array - supports multiple spots)
  touristSpots: [
    {
      spotId: "507f1f77bcf86cd799439011",  // Trek spot MongoDB ID
      name: "Rampa Falls",                  // Trek spot name
      visitDate: "2026-02-15T00:00:00.000Z", // Visit date
      
      // Counts
      counts: {
        guests: 4,    // Number of guests
        cameras: 2    // Number of cameras
      },
      
      // Calculated Amounts
      amounts: {
        entry: 400,   // 4 guests × ₹100 = ₹400
        camera: 100,  // 2 cameras × ₹50 = ₹100
        parking: 0,   // Not used
        addOns: 0,    // Not used
        total: 500    // Entry + Camera = ₹500
      },
      
      addOns: []      // Not used
    }
    // ... more spots if multiple selected
  ],
  
  // Payment Information
  totalPayable: 500,              // Grand total (sum of all spots)
  status: "reserved",             // reserved | pending | not-reserved | cancelled
  paymentStatus: "paid",          // paid | unpaid | cancelled
  paymentTransactionId: "",       // Optional transaction ID
  
  // User/Customer Details
  user: {
    name: "John Doe",
    email: "john@example.com",
    phone: "9876543210",
    address: "123 Main Street",
    address2: "Apartment 4B",
    city: "Hyderabad",
    state: "Telangana",
    pincode: "500001",
    country: "India"
  },
  
  // Booking Metadata
  reservationDate: "2026-02-09T00:00:00.000Z",  // When booking was made
  bookingDate: "2026-02-09T00:00:00.000Z",      // Same as reservationDate
  reservedFrom: "Admin",                         // Admin | Online | Phone | Walk-in
  userId: null,                                  // Link to User model if exists
  
  // Timestamps (Auto-generated by MongoDB)
  createdAt: "2026-02-09T10:30:00.000Z",
  updatedAt: "2026-02-09T10:30:00.000Z"
}
```

---

## 📊 Breakdown by Section

### 1. Trek Spot Information
**What's saved:**
- Trek spot ID(s)
- Trek spot name(s)
- Visit date

### 2. Guest & Camera Counts
**What's saved:**
- Number of guests (e.g., 4)
- Number of cameras (e.g., 2)

### 3. Calculated Fees (Per Spot)
**What's saved:**
- Entry fees = guests × spot.entryFees
- Camera fees = cameras × spot.cameraFees
- Spot total = entry + camera

### 4. Grand Total
**What's saved:**
- Sum of all spot totals
- Example: If 2 spots selected, each ₹500 = ₹1000 total

### 5. User Details
**What's saved:**
- Full name
- Phone (required)
- Email (required)
- Complete address (address1, address2, city, state, postal code, country)

### 6. Booking Status
**What's saved:**
- Booking status (reserved/pending/not-reserved/cancelled)
- Payment status (paid/unpaid/cancelled)
- Reservation date (when booking was created)
- Reserved from (Admin/Online/Phone/Walk-in)

## 🔢 Calculation Example

### Form Input:
- Trek Spots: Rampa Falls, Maredumilli Waterfalls
- Guests: 5
- Cameras: 3
- Visit Date: 2026-02-20

### Rampa Falls Pricing:
- Entry Fee: ₹100 per guest
- Camera Fee: ₹50 per camera

### Maredumilli Waterfalls Pricing:
- Entry Fee: ₹80 per guest
- Camera Fee: ₹40 per camera

### Calculation:

**Rampa Falls:**
- Entry: 5 guests × ₹100 = ₹500
- Camera: 3 cameras × ₹50 = ₹150
- Spot Total: ₹650

**Maredumilli Waterfalls:**
- Entry: 5 guests × ₹80 = ₹400
- Camera: 3 cameras × ₹40 = ₹120
- Spot Total: ₹520

**Grand Total: ₹1,170**

### Saved Data:
```javascript
{
  touristSpots: [
    {
      name: "Rampa Falls",
      counts: { guests: 5, cameras: 3 },
      amounts: { entry: 500, camera: 150, total: 650 }
    },
    {
      name: "Maredumilli Waterfalls",
      counts: { guests: 5, cameras: 3 },
      amounts: { entry: 400, camera: 120, total: 520 }
    }
  ],
  totalPayable: 1170
}
```

## ✅ Summary

**Yes, we are saving:**
- ✅ Guest count
- ✅ Camera count
- ✅ Guest fees (entry fees)
- ✅ Camera fees
- ✅ Per-spot totals
- ✅ Grand total

**We are NOT saving:**
- ❌ Internal notes (removed)
- ❌ Vehicle type/count (removed)
- ❌ Parking fees (removed)

## 🔍 Where to Find This Data

**Database:** MongoDB collection `touristspotreservations`

**API Endpoint:** `GET /api/tourist-booking/all-bookings`

**Admin Panel:** Tourist Bookings list page

---

## 📝 Notes

1. **Multi-spot support**: The system supports booking multiple trek spots in a single booking
2. **Price snapshot**: Prices are saved at booking time, so future price changes don't affect existing bookings
3. **Auto-calculation**: Backend recalculates all amounts to prevent tampering
4. **Validation**: Backend validates that selected spots exist and have valid pricing
